Introdução
N.T.: Boa parte das piadas sem graça do documento original foram suprimidas para
facilitar a tradução.
N.T.: O vi é um editor de texto padrão do mundo UNIX, podendo ser encontrado em
qualquer distribuição do Linux. No entanto, o seu uso é bastante diferente de
editores de textos do mundo Windows. Essa foi a principal motivação para efetuar a
tradução desse tutorial. As únicas partes essenciais desse texto são as duas primeiras
seções. Mas, é bastante desejável que um adminstrador de sistemas UNIX tenha um
domínio mais amplo de uma ferramenta tão útil e importante como o vi. Por isso,
recomendo a todos que pelo menos dêem uma lida até o final.
Vi? V o quê?
Vi é um "editor visual" - um editor que lhe permite visualizar um documento, na sua
totalidade, a medida que você o cria e edita. Aqueles que pertencem à geração
Microsoft Word podem achar isso um pouco redundante - mas é claro, você está
pensando, todos os editores deixam você fazer isso! Bem, não exatamente - nos
primórdios do UNIX, editores de linha como edlin e ex eram o padrão e criar um
documento em uma dessas belezinhas envolvia o uso de comandos arcanos para
adicionar, remover e recuperar o texto de um buffer invisível.
O tempo passa e quase todos os Unices hoje vem com alguma variante do vi - ele é,
com certeza, um dos editores mais poderosos por aí, com uma grande quantidade de
características úteis e interessantes. Eu costumo chamá-lo de Ferrari dos editores de
texto: é rápido, tem estilo e uma vez que você sabe como dirigir você nunca mais
vai ter dificuldade de arrumar um encontro no sábado à noite...
Iniciando
Iniciar uma sessão do vi é ridiculamente simples. Digite
$ vi
no prompt do terminal UNIX, seguido de um enter, e o vi abrirá um arquivo vazio
para você começar a trabalhar. Alternativamente, você pode especificar o arquivo a
ser aberto na própria linha de comando, dessa forma:
$ vi meuarquivo
Não se preocupe com os no editor - eles são apenas indicadores visuais do fim do
seu arquivo e não aparecem no seu documento.
A primeira (e mais confusa) coisa sobre o vi são os vários diferentes modos nos
Page 2
quais o editor pode operar. Para os nossos propósitos, eu vou separá-los em duas
amplas categorias: "modo de edição" (insert mode) e "modo de comando" (command
mode). O "modo de edição" é aonde você vai passar a maior parte do seu tempo -
ele permite adicionar (ou inserir) texto no seu documento. O "modo de comando", por
outro lado, permite que você se mova pelo documento, apague ou substitua blocos de
texto e acesse funções úteis do vi como buffers, por exemplo.
Quando o vi é iniciado, ele geralmente está em modo de comando. Para entrar no
modo de edição digite
i
o vi mostrar um aviso da troca de modos no canto inferior esquerdo. Agora você pode
digitar o texto no documento de maneira normal. Quando tiver terminado, aperte
ESC
para retornar ao modo de comando.
Para começar a inserir texto numa linha nova, use
o
para entrar no modo de edição com o cursor já posicionado na próxima linha, ou
O
para entrar no modo com o cursor na linha anterior.
Além de inserir texto, o vi também permite que você sobrescreva um texto existente
- isso é chamado de "modo de substituição" (replace mode), que pode ser acessado
digitando-se
R
no modo de comando. Tudo que você digitar agora será "escrito por cima" do texto
existente. Se a sua intenção era substituir um único caractere - por exemplo, para
corrigir um erro de digitação - use
r
para substituir um único caractere; o editor retornará automaticamente para o modo
de comando quando o caractere for substituído.
Obviamente, em algum momento, você vai precisar salvar o seu trabalho. No vi, isso
é feito via o comando de "escrita" (write), abreviado para
:w
Para sair, use o comando
:q
O vi também permite que se combine comandos. Por exemplo, tente
:wq
Page 3
para salvar seu documento e sair do programa. Alternativamente, se você quiser
descartar seu texto use
:q!
que sai do vi sem salvar nenhuma alteração do arquivo sendo editado.
Obviamente, todos esses comandos com ":" devem ser executados no modo de
comando. Tentar utilizá-los no modo de edição os fará aparecer como parte do seu
documento.
Vamos dançar!
Agora que você já sabe editar um texto, está na hora de ensiná-lo a segunda coisa
mais confusa do vi: se movimentar pelo documento. Para aproveitar melhor essa
lição, eu sugiro que você abra um documento já existente ou crie um novo e digite
algumas linha nele. Certifique-se que você está no modo de comando teclando
ESC
algumas vezes quando tiver terminado.
As teclas de movimento do vi são:
h = esquerda,
j = baixo,
k = cima,
l = direita.
Em alguns consoles UNIX, também pode ser possível usar as setas do teclado para se
movimentar pelo documento; entretanto, eu sugiro que você tente memorizar a lista
acima pois elas vão funcionar em qualquer situação.
Gostaria de se mover entre as palavras? Use
w
para mover uma palavra para frente, ou
b
para mover uma para trás.
Para se mover entre parágrafos, use
{
e
}
O movimento entra sentenças requer o uso cuidadoso de
(
e
Page 4
)
Se você precisa se mover até uma linha específica do seu documento - linha 568, por
exemplo - use esse comando:
:568
Para ir para o início do arquivo, digite
gg
e para o fim, somente
G [isso é shift+G]
E o vi também tem um equivalente a "Page Up e "Page Down". Use
^F [isso é ctrl+F]
para ir uma página a frente, e
^B [isso é ctrl+B]
para uma página atrás.
Para saber a sua posição no documento, use
^G [isso é ctrl+G]
É claro que essas não são as únicas teclas de movimento - existem muitas outras. No
entanto, por enquanto, essas devem ser suficientes para o seu próximo exercício.
Cortar, Copiar, Colar...
Agora vamos ver como apagar e copiar textos. Para apagar um único caractere,
posicione o curso sob ele, certifique-se que você está no modo de comando, e tecle
x
para apagá-lo. Para apagar uma palavra, tecle
dw
que é a abreviação de "delete word" (apagar palavra). Para apagar uma linha inteira
use
dd
Já escritores com bloqueio criativo vão apreciar a rapidez dos comandos
dgg
e
dG
que apagam tudo da posição corrente do cursor até o começo e fim do arquivo
Page 5
respectivamente.
Oops... você não usou esses dois, não é? Não tem problema, o vi também possui
um comando de "undo" (desfazer) muito útil, que pode ser usado teclando
u
sempre que você precisar desfazer algum engano. E é claro, junto com o undo temos
o comando de "redo" (refazer), que permite repitir ações passadas. Basta teclar
. [isso mesmo, um ponto]
Quando você apaga alguma coisa no vi, o texto apagado geralmente vai parar num
buffer temporário na memória, aonde ele fica até ser substituído. Esse texto pode ser
re-inserido no seu documento com o comando de "put" (colocar). Se você estiver
acompanhando tudo já deve imaginar que o comando de "put" é acessado com
p
Vamos lá, tente. Apague uma linha, mova o cursor para um novo parágrafo e a
coloque em outro lugar. Simples, e equivalente à funcionalidade cortar e colar que
existe em outros editores.
Caso você queira inserir o texto apagado acima da posição atual do cursor,
simplesmente use
P
ao invés de
p
E quanto a copiar e colar? Também é muito fácil: o vi usa o comando "yank" para
copiar um texto de uma forma bem similar ao comando de apagar. Para copiar uma
palavra, use
yw
Já
yy
copia uma linha inteira e a coloca no buffer temporário, pronta para ser colocada em
outro lugar.
Um, dois, três...
Uma das funções mais poderosas do vi é repetir automaticamente um comando um
número específico de vezes, bastando preceder o comando com um número. Por
exemplo, suponha que você precisa apagar oito linhas de um documento. Baseado no
que você já aprendeu, você usaria o comando
dd
oito vezes, criando essa linha de comando horrivelmente grotesca:
Page 6
dddddddddddddddd
O que você poderia ter feito, caso eu já tivesse dito, era
8dd
Como você verá ao usá-lo, esse comando simplesmente executa o comando "dd"
oito vezes, apagando oito linhas do documento sem que você precise ir a uma
farmácia comprar uma aspirina depois. E essa técnica pode ser usada com quaisquer
dos comandos que você já aprendeu: você pode utilizá-la para copiar múltiplas
linhas, repetir inserções um número específico de vezes, e até mesmo mover o cursor
para ocorrências específicas de palavras no documento.
Procurando por hope
O vi também possui funcionalidades de busca e substituição bastante poderosas, que
podem se acessadas com os comandos / e ?.
Digamos que você escreveu um tratado sobre o papel interpretado por Mark Hamil
em "Star Wars: A New Hope" (Guerra nas Estrelas: Uma Nova Esperança) - mas, ao
revisar o seu documento, você vê que digitou erroneamente a palavra "hope"
(esperança) como "dope" (narcótico) em diferentes locais do texto. Obviamente você
não pode ignorar esse erro: isso levaria a uma interpretação completamente nova do
filme de ficção-científica mais querido do planeta. O que você faz?
/dope
O comando /padrão diz ao vi para procurar o documento pelo padrão especificado;
ao encontrá-lo, ele vai posicionar o cursor no primeiro caractere da palavra, e
aguardar novos comandos.
Para buscar para trás, use o comando ?padrão, assim:
?dope
É improvável, até mesmo tolice da sua parte, assumir que esse erro só ocorreu uma
vez. Você pode repertir a última busca pressionando
n
ou, na língua do vi, "next match" (próxima ocorrência).
Não é suficiente somente identificar o seu erro. O próximo passo, conforme a
maioria das doutrinas religiosas e pastores da televisão, é consertá-lo e se certificar
de que ele não vai acontecer de novo. O vi também pode ajudá-lo nisso também,
ainda que isso involva memorizar um comando que, a primeira vista, provavelmente
vai te deixar perguntando se Deus existe mesmo...
:1,$s/padrão_a_encontrar/padrão_para_substituir/g
No caso de você estar se perguntando de onde o "s" veio, é a abreviação do
comando "substituir". O 1 e o $ são os símbolos que indicam a faixa onde a
substituição deve ocorrer - nesse caso, do início do documento [linha 1] até o fim
[$]. O "g" é um flag que indica que todas as ocorrências devem ser substituídas.
Page 7
Logo, no exemplo acima, o comando para substituir a palavra "dope" por "hope" é
:1,$s/dope/hope/g
Se pelo menos conseguir o número do telefone da Leia fosse tão fácil assim...
Bookmarks, Buffers...
O vi também permite que você defina "bookmarks" (marcadores) no seu documento
para facilitar o salto entre seções específicas de arquivos grandes. Como um exemplo,
considere os seguintes blocos de texto:
...este documento delineia uma proposta para a estrutura corporativa da Carros
Potentes Ltda, uma companhia no negócio de produção e venda... [página 2]
...o presidente da Carros Potentes Ltda, James van Hausen, que, em sua adolescência
desvairada, era muito conhecido por derrubar árvores usando caríssimos carros
esporte, conseguiu... [página 137]
E então, vamos colocar alguns bookmarks? Na página 2, mova o cursor até a
palavra "documento", e digite
ma
Isso coloca um bookmark, identificado pela letra "a", na posição do cursor. Agora vá
até a página 137 e adicione um bookmark chamado "b" na palavra "carros". Para
voltar para a página 2, tudo que você precisa fazer é digitar
a
e para chegar à página 137, simplesmente digite
b
Qualquer letra minúscula pode ser usada como um identificador de bookmark.
Obviamente, você está limitado a um total de 26 bookmarks por arquivo - mas, na
maoiria das vezes, isso é mais que suficiente. Se você precisar de mais que isso eu
sugiro fortemente que você faça terapia.
Além dos múltiplos bookmarks, o vi também possui múltiplos buffers. Se você
esteve prestando atenção, sabe que todo o material que você apaga ou copia vai para
um buffer temporário até ser substituído ou removido. Mas o vi também possui
"buffers com nome" - como se fossem armários, esses buffers podem ser usados para
guardar diferentes blocos de texto, deixando-os disponíveis para inserção aonde
forem necessários.
Como um exemplo, dê uma olhada nesse pedaço de um texto que me foi entregue por
uma de minhas ex-alunas:
...coelinhas (N.T.: da Playboy americana) parecem ter uma vida muito boa. Tudo o
que elas fazem é ficar na praia, bebendo pina coladas e passando bronzeador. De vez
em quando, elas se levantam e jogam um pouco de voleyball. Parece divertido - aonde
eu assino?...
Tenho certeza que vocês entendem porque eu falei "ex-aluna"...
Page 8
Agora vamos supor que eu quisesse trocar de posição as frases "bebendo pina
coladas" e "passando bronzeador". Isso é o que eu faria:
1. Mover o cursor para o começo da palavra "bebendo"
2. Apagar as três palavras e movê-las para um buffer chamdo "p"
"p3dw
3. Mover o cursor para o começo da palavra "passando"
4. Apagar essas duas palavras e movê-las para um buffer chamado "s"
"s2dw
5. Mover o cursor de volta para antes da palavra "e"
6. Inserir o conteúdo do buffer "s"
"sp
7. Mover o cursor para depois da palavra "e"
8. Inserir o conteúdo do buffer "p"
"pp
9. Me parabenizar espalhafatosamente, como eu estou fazendo agora!
Como você pode ver, para usar um buffer, basta preceder o comando normal de
copiar/apagar/substituir com o identificador do buffer. Um identificador de buffer
consiste de uma aspas dupla seguida por uma letra minúscula. No exemplo acima,
tanto "s quanto "p são identificadores de buffers.
Devo lembrá-los aqui que nem os bookmarks, nem os buffers são mantidos quando
você sai do editor.
Sobre arquivos e janelas
Você já viu que
$ vi nome_do_arquivo
inicia o vi com o arquivo especificado já carregado. Mas por que se restringir a um?
Você pode carregar múltiplos arquivos desta forma:
$ vi arquivo1 arquivo2 arquivo3
Para alternar entre os arquivos use esses comandos:
:next = vai para o próximo arquivo na lista
:rewind = vai para o primeiro arquivo na lista
:last = vai para o último arquivo na lista
Para sair de todos os arquivos de uma vez use:
:qa!
Page 9
para sair sem salvar, ou
:wqa
para salvar todas as alterações e sair.
Após já ter iniciado o vi, você pode carregar um novo arquivo no editor com o
comando "editar", assim:
:e /caminho/para/arquivo.txt
Se você preferir inserir o conteúdo de outro arquivo diretamente no documento que
você está editando agora existe um comando de leitura "read" feito justamente para
isso. Tente:
:read /caminho/para/arquivo_a_ser_inserido.txt
Se você é um programador, definivamente vai gostar da funcionalidade do vi de te
mostrar múltiplas visões do mesmo arquivo, que no jargão computacional é
chamado de "window splitting" (divisão de janelas). E o comando é, naturalmente
:split
Cada uma das janelas criadas pode ser manipulada de forma independente das outras,
apesar das mudanças feitas no arquivo ficarem imediatamente visíveis em todas elas.
Vamos avançar um pouco mais. Que tal carregar arquivos diferentes em cada uma das
janelas? Por exemplo, não seria ótimo se você pudesse carregar um arquivos HTML
em uma, um script CGI associado em outra, alguma documentação de ajuda numa
terceira e ter todas as três visíveis ao mesmo tempo?
Bem, você pode. Basta usar o comando "new" (novo) para criar uma nova janela para
cada um dos arquivos, assim:
:new form.html
:new mailform.cgi
:new help.txt
Para alternar entre as janelas, use
^W W [isso é ctrl-W seguido por um W]
